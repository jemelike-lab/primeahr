## Architecture patterns to remember

### Bridge handler shape (Phase D)

Each handler file exports a `RegistryEntry`:
```typescript
const handlerMap: Record<string, EventHandler> = {
  'event.type': handlerFn,
  // ...
};
export const xxxHandlers = {
  resolve(eventType: string): EventHandler | null {
    return handlerMap[eventType] ?? null;
  },
  knownEventTypes: () => Object.keys(handlerMap),
};
```
Each handler function returns `EventHandlerResult`:
```typescript
{ target_table, target_id, outcome: 'success'|'failure'|'partial'|'requires_review'|'ignored', result_summary, notes? }
```
Do NOT export `Record<string, EventHandler>` directly — `inbox.ts` REGISTRY needs the wrapper.

### VPS chunked transfer (for files >5KB when SCP/PAT not available)

```
tar czf file.tar.gz . → base32 -w0 → split -b 1800 → cat each chunk
→ printf '%s' '<real chunk content>' >> /tmp/file.b32
→ md5-verify cumulative every 2-4 chunks
→ base32 -d /tmp/file.b32 > /tmp/file.tar.gz → md5-verify → tar xzf
```

Use base32 not base64 (xterm.js collapses `//` to `/`). ALWAYS `cat /tmp/cNN` to get real chunk content — never fabricate from context.
